CodeMap × D2 — 风格方案对比

三个画布共享同一份子图（4 个 class · 6 个 method · 1 个 external dep · 5 条 call 边） prototype · 决定后删除
A

完全 D2 化（浅色纸张）

暖白米色画布 + 圆角矩形 + 灰蓝描边 + 衬线字体。视觉冲击最强，画布与暗色工具栏形成"明信片"对比。

B

仅画布 D2 化（推荐起点）

工具栏 / outline 保持暗色，只把画布换成 D2 浅色 + 圆角 + 折线。读图体验靠近 D2 文档，但扩展整体风格延续。

C

暗色 D2 风（最小幅度）

保留暗色背景，只挪用 D2 的形状语言：圆角矩形、柔和描边、更厚的描边权重、折线边、降低饱和的填充色。

A · 完全 D2 化

  • 背景 #F4F1EA
  • 节点填充 #E8E5DC
  • 描边 #7A8B99
  • 字体：Source Sans / 衬线
  • 边：直角折线，灰蓝色
  • 最像 D2 文档
  • ⚠ 与 VS Code 暗色 UI 风格反差大
  • ⚠ 高亮 / unverified 等状态色都要重新设计（暗色配色不能直接用）

B · 仅画布 D2 化

  • 外壳暗色不变，仅 #cy 区域换 D2 主题
  • 需要重做状态色映射（risk/verification 在浅色画布上的对比度）
  • minimap / overlay 元素的颜色要相应调整
  • ✓ 读图区域 D2 感强，整体扩展风格不分裂
  • ✓ 改动局部，回归面小
  • ⚠ 浅 / 暗交界处需要边框过渡

C · 暗色 D2 风

  • 背景 / 字体保持，只换节点形状与边样式
  • 挪用 D2 的：圆角度数、描边权重、折线边、柔化填充
  • 状态色 (capture/recall/host/shared/risk/verification) 几乎不动
  • ✓ 改动量最小，零回归风险
  • ✓ 不需要重做颜色体系
  • ⚠ 视觉差异最不明显——更像"现有风格 polish"，不是"D2 化"
